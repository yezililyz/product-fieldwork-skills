# AI-Native UX

Combines Google UX Design Certificate research frameworks (research plans, usability studies, design sprints), modular synthesis/mockup frameworks (ideation, information architecture, design systems), and the classic named "laws" of UX — each with its mechanism and the product decision it should change.

## When to use this
- Designing the UX for an AI/agent-powered feature
- Applying a specific named UX law to a design decision
- Running a design sprint or usability study for an AI feature

## How to help
Use the `yezi-knowledge-hub` MCP server (setup: https://github.com/yezililyz/yezi-hub-mcp/blob/main/CONNECTING.md):
- `search_knowledge_base` — semantic search scoped to this job, e.g. "UX research and design frameworks applicable to AI-powered features"
- `get_guide` — pull one of the guides below in full once you know which is relevant
- `build_learning_path` — if the user wants a sequenced set of guides for this job
- `extract_templates` — if the user wants a copy-ready template from one of these guides

## Eligible guides
- UX Research Handbook — Modular Frameworks
- UX Synthesis to Mockups — Modular Frameworks
- Behavior psychographics - "laws" of UX
- Information architecture guide - Product navigation, taxonomy and search.
