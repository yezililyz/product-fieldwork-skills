# User Onboarding & Activation

Fifteen named onboarding design patterns (each with when to use it, the UI elements it needs, and real app examples), data from studying over a thousand onboarding flows that challenges the assumption that shorter is always better, and the EUREKA product-led-onboarding framework fused with real AI-industry GTM notes.

## When to use this
- Designing a first-run flow for a new product or feature
- Diagnosing a specific activation-funnel drop-off
- Picking the right named onboarding pattern for a given product type
