# Engineering Health

A reusable debugging system built on the insight that a project that "keeps breaking" usually has the same handful of missing foundations spawning every new bug — build the net once, then run one ritual per change — paired with a system-design framework of scoping first, deriving requirements from the user, then building and defending the architecture.

## When to use this
- Diagnosing recurring bugs that keep coming back in different forms
- Building a reusable debugging discipline instead of firefighting each bug alone
- Applying a scope-first system-design lens to an engineering process problem
