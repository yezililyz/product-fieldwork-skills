# Data Infrastructure

Covers the full technical foundation: feature stores solving training-serving consistency (with real workflows and use cases), the Six Pillars of data security (Governance, Discovery, Protection, Compliance, Detection, Response), the jump from a one-off data project to a trusted, continuously-released data product, ML-specific attack surfaces and compliance obligations, four operating disciplines for data quality/human judgment/workflow-mapping/outcome-first design, real deployments at a major bank/Walmart/General Motors for the data-quality-vs-liability trade-off, data lineage's two operating levels, 61 real data-observability use cases from BlaBlaCar/Mercari/Vimeo/Checkout.com/Contentsquare, and API design judgment from PMs who built at PayPal, Square, Twilio, and Stripe.

## When to use this
- Designing a feature store or data pipeline for ML
- Explaining data lineage, quality, or governance to a non-technical stakeholder
- Designing or reviewing a public/internal API to a Twilio/Stripe-caliber standard
- Deciding what data security/compliance controls a new system actually needs
- Diagnosing a data-trust or data-observability problem using real deployment examples
