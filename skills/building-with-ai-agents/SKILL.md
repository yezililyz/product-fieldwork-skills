# Building With AI Agents

Covers the real mechanics: architecture choices between API vs. browser agents, the MCP protocol layer that lets a model reach past the chat window into your inbox/database/tools, memory systems and guardrails that turn random generation into reliable agent behavior, parallelization and multi-agent debate patterns from a $4M+ agency playbook, and the frameworks/runtimes for building agentic systems on Google Cloud specifically.

## When to use this
- Architecting a multi-step agent or agentic workflow
- Deciding between a single LLM call vs. a full agent loop
- Giving an agent tool access (MCP) or memory/guardrails so it's reliable, not just generative
- Automating a business process end-to-end with AI
- Picking a copy-ready prompt to start from instead of writing one from scratch
