# Writing PRDs

A teardown of Linear's real "priority micro-adjust" PRD — a small feature (drag-and-drop reordering within a priority bucket) written as a masterclass: it reframes the feature request as a problem, solves it in the most on-brand way possible, and says no to more powerful options on principle.

## When to use this
- Writing a PRD for a scoped, shippable change
- Reviewing someone else's PRD for scope creep or unexamined assumptions
- Learning from a real, sentence-by-sentence dissected example PRD
